# 📚 Online Book Store SQL Analysis

## 📌 Project Overview
This project demonstrates SQL-based data analysis on an Online Book Store dataset. The objective is to perform data exploration, business analysis, and reporting using SQL queries on multiple related tables.

The project uses three CSV datasets:

- Books.csv
- Customers.csv
- Orders.csv

These tables are connected through common columns such as `Book_ID` and `Customer_ID`, enabling relational analysis and business insights.

---

## 🎯 Objectives

- Practice SQL fundamentals and advanced queries
- Perform data analysis on bookstore sales data
- Analyze customer purchasing behavior
- Generate revenue and inventory insights
- Understand relationships between multiple tables

---

## 📂 Dataset Structure

### Books Table
Contains information about books available in the store.

**Columns:**
- Book_ID
- Title
- Author
- Genre
- Price
- Published_Year
- Stock

### Customers Table
Contains customer information.

**Columns:**
- Customer_ID
- Customer_Name
- Country
- City

### Orders Table
Contains order transaction details.

**Columns:**
- Order_ID
- Customer_ID
- Book_ID
- Quantity
- Order_Date
- Total_Amount

## 📊 SQL Analysis Performed

### Basic Queries

✔ Retrieve all books in the Fiction genre

✔ Find books published after 1950

✔ List customers from Canada

✔ Show orders placed in November 2023

✔ Calculate total stock available

✔ Find the most expensive book

✔ Identify customers ordering multiple quantities

✔ Retrieve orders exceeding $20

✔ List all available genres

✔ Find the book with lowest stock

✔ Calculate total revenue generated

---

### Advanced Queries

✔ Total books sold by genre

✔ Average price of Fantasy books

✔ Customers with at least 2 orders

✔ Most frequently ordered book

✔ Top 3 most expensive Fantasy books

✔ Total quantity sold by author

✔ Cities with customers spending over $30

✔ Highest spending customer

✔ Remaining stock after fulfilling orders
